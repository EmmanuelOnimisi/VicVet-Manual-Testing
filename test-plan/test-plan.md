# Test Plan – VicVet Clinic Management System

## 1. Introduction

This test plan describes the manual testing approach for the VicVet Clinic Management System. The purpose is to verify that the application's features function correctly and meet the expected requirements.

## 2. Objectives

- Verify that all core features work correctly.
- Identify defects before release.
- Ensure the application provides a reliable user experience.

## 3. Scope

### Features to be Tested

- User Registration
- User Login
- Dashboard
- Pet Management
- Owner Management
- Appointment Management
- Medical Records
- Staff Management

### Features Not Tested

- Performance Testing
- Security Testing
- Load Testing

## 4. Testing Type

- Manual Testing
- Functional Testing
- Smoke Testing
- Regression Testing

## 5. Test Environment

- Operating System: Windows 11
- Browser: Google Chrome
- Database: MySQL
- Local Server: XAMPP

## 6. Entry Criteria

- Application installed successfully.
- Database configured.
- Test data available.

## 7. Exit Criteria

- All critical test cases executed.
- High-priority defects resolved or documented.

## 8. Deliverables

- Test Plan
- Test Cases
- Bug Reports
- Test Summary Report
